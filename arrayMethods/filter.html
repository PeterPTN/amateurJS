Filter() method

The filter() method returns a new array of elements that pass a boolean condition.

This condition is nested within a function you provide.

We will store this new array in numbersDivisableByTwo.

Here we call filter() after our calling array numbers.

Within filter's () we add our function. In this function we only return numbers that, when divided by 2, leaves a remainder of 0.

Here our function is called 4 times -- once for each array element. number's value changes to the next array element with every new function call.

Each function call will essentially test these conditions:

10 % 2 === 0 | 32 % 2 === 0 | 67 % === 0 | etc.

If true then the value is added to the new array.

                    
const numbers = [10, 32, 67, 99];

const numbersDivisableByTwo = numbers.filter(
    function(number) {
        return number % 2 === 0
    });

console.log(numbersDivisableByTwo);

// Expected Output: Array [10, 32]

Tangents

Methods are functions but simply relate to an object - in this case the Array object.

Essentially filter() is a function which requires another function (that you provide) to work.

Typically you will see anonymous functions and other short-hand variations: 

.filter((number) {
    return number % 2 === 0;
});

// or

.filter(number => number % 2 === 0);

They both do the same thing.

This method is appropriate when you need a larger, more general data set into something more specific.

Another Use Case

Often data is stored within an array of objects. Occasionally our data might not be well maintained. Our staff array is an example of that.

For whatever reason we have 2 directors. Let's say that Bob Jane is this company's director.

Let's imagine this array has +50 elements.

We want to be able to store our director's information from our staff array to our director variable so that we can easily access his name and department whenever we like.

Like above, we .filter (and loop) over our existing staffs array.

Again we have a provided function. Here we outlined 2 conditions due to the error in our staff array listing 2 directors.

The conditions will only return the array element (in this case an Object) where name is equal to Bob Jane and where department is equal to Director.

We can now use the director variable whichever way we want.


const staff = [
    {
        name: "Bob Jane",
        department: "Director"
    },
    {
        name: "Lindsey Wagner",
        department: "Director"
    }
];

const director = staff.filter((staffMember) => {
    return staffMember.department === "Director" & staffMember.name === "Bob Jane";
});

console.log(director);

/* 
Expected Output: Array [
    {
        name: "Bob Jane",
        department: "Director"
    }
];
*/

Tangents

Use cases like this aren't uncommon. Typically you will encounter them when fetching data from an API.

Sometimes it's much easier to simply count which index the element you need is and set it to a variable.

const realDirector = staff[0];

Other times the array is far too big and counting indexes isn't prudent

Instead you could use something like this to find the right index.

const directorIndex = staff
    .map(staffMember => staffMember.name)
    .indexOf("Bob Jane");

console.log(directorIndex);

// Expected Output: 0

const director = staff[directorIndex];

But this method only checks for a single condition which may or may not be enough for certain use cases.
